# 更新日志

按时间倒序排列。版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## v0.3.0 — 2026-06-03

### 新增
- **数字列仿射变换**：用户可手动勾选数字列，自动应用 `y = a·x + b` 的可逆变换，
  每列独立随机 (a, b)。绝对量级被掩盖，但比例、增长率、相关性、求和都精确，
  AI 算出的数值能精确还原。
- **日期列项目级偏移**：所有 datetime 列自动统一偏移随机天数（±5 年内），
  保留时间间隔、周期、星期、季节性。新建项目自动生成偏移；旧项目首次使用时
  按需生成（向后兼容）。
- **GUI 列分类显示**：
  - 文本列：可勾选，可填前缀
  - 数字列 `[数字]`：可勾选，前缀框禁用（自动仿射变换）
  - 日期列 `[日期·自动偏移]`：勾选锁定（自动处理）
- **更详尽的 prompt 模板**：明确告知 AI 哪些列是文本占位、哪些是仿射数值、
  哪些是偏移日期，便于 AI 给出准确分析。
- **版本号管理**：`core/version.py` 单一来源；GUI 标题、打包文件名、CI artifact
  名都自动带版本号。

### 修复
- 多 sheet 同列名场景下，仅在第一行填前缀其他留空导致的"两个不同前缀"误报
  （上一版已修，此版本继续优化 UI）。

### 已知限制
- 仿射 + 日期偏移防不住形状指纹攻击。如担心专业分析师对照公开财报，需额外加噪
  （未实现）。

## v0.2.0 — 2026-06-02

### 新增
- **多 sheet 同列名去重**：同一列名在多个 sheet 中只显示一行，标注"出现于 N 个 sheet"，
  填一次前缀对所有 sheet 生效。
- **默认勾选非数值列**：打开文件后所有文本列自动勾选，数字列保持不勾。
- **顶部黄色提示横幅**：提醒用户检查勾选并尽量多脱敏。
- **密码最低 8 位强制检查**：避免弱密码被离线 GPU 爆破。
- **数字列警告**（已被 v0.3.0 替换为正式的仿射变换支持）。
- **README 醒目说明**："脱敏文件 ≠ 加密文件"。

### 修复
- `core/encrypt.py` 中重复的 `Optional` import。
- Tokenizer 不再禁止"不同列名共享同一前缀"——这是合理需求，比如旧文件叫"游戏名称"、
  新文件叫"游戏名"想都用 `GAME` 让"原神"得到同一 token。
- Prompt 模板措辞从"加密"改为更准确的"脱敏"。

## v0.1.0 — 2026-06-01

### 初始版本
- 核心脱敏 / 还原流程（文本列 → token，相同值同 token，跨文件项目级一致）
- AES-128 + HMAC + PBKDF2 加密的项目密钥文件
- PyQt6 GUI（脱敏 / 还原 / 历史 三个 tab）
- 支持 .xlsx / .xls / .csv / .tsv
- 还原宽容匹配：容忍大小写、空格、引号、末尾标点；AI 编造 token 标记为 `[⚠未识别]`
- 自动生成给 AI 的 Prompt 模板（不泄露任何原值）
- GitHub Actions 自动 Win/Mac 双平台打包
