# Enigma — 本地数据脱敏工具

把表格里的敏感字段替换成占位符，交给 ChatGPT/Claude 处理后再一键还原。
**所有过程纯本地，密钥不出本机。**

## 三种脱敏模式

| 列类型 | 脱敏方式 | 还原 |
|---|---|---|
| **文本列**（游戏名、玩法等） | 替换为 `GAME_0001` 这类占位符；同值同 token | 字符串精确还原 |
| **数字列**（流水、DAU 等，需手动勾选） | 仿射变换 `y = a·x + b`，每列独立随机 (a, b) | 数值精确还原（含浮点） |
| **日期列**（自动检测） | 项目级统一偏移随机天数 | 日期精确还原 |

数字 / 日期脱敏的设计目的：即便 AI 看到了流水数字和日期，也无法通过"绝对量级"和"绝对时间点"反推具体公司或游戏。

## 安装与运行（开发模式）

```bash
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

## 打包成可执行文件

```bash
pip install pyinstaller
python build/build.py
# 产物在 dist/ 目录，文件名带版本号
```

## 工作流

1. **新建项目** → 选择 `.keyfile` 保存位置 → 设置密码（密码丢了无法还原，请记牢）
2. **脱敏**：选数据表 → 勾选要脱敏的列（默认勾选所有非数值列）→ 一键脱敏
3. 把生成的 `*_encrypted.xlsx` 上传给 AI；同时复制工具生成的 Prompt 模板贴到对话开头
4. **还原**：把 AI 处理完的文件喂回工具 → 一键还原

## 文件目录

```
项目所在目录/
├─ myproject.keyfile      # 加密的密钥文件（核心）
├─ encrypted/             # 工具产生的脱敏后文件
└─ decrypted/             # 工具产生的还原后文件
```

## 安全说明

> ⚠️ **重要：脱敏文件 ≠ 加密文件**
>
> `encrypted/` 目录下的文件是**已脱敏的明文**，不是加密文件。
> 它包含真实的非敏感字段（未勾选的列）和占位符 / 仿射后的数值 / 偏移后的日期。
> 可以放心给 AI，但**不应公开发布**。真正受密码保护的只有 `.keyfile`。

- 密钥文件用 AES-128 + HMAC（Fernet）+ PBKDF2-HMAC-SHA256（48 万次迭代）加密
- 密码至少 8 位（弱密码可被离线 GPU 爆破）
- 工具不发起任何网络连接
- 跨文件一致：同一项目内，相同的原值得到相同的 Token；同一数字列共用同一组 (a, b)
- 还原阶段宽容匹配：容忍大小写、空格、引号、末尾标点
- AI 编造的不存在 Token 会被标记 `[⚠未识别]` 而不是误还原

## 已知限制

仿射变换 + 日期偏移 **不能完全防止形状指纹攻击**——如果攻击者有公开的真实数据
做时间序列对照，趋势的形状（暴涨、衰退）仍然可以匹配。这是"既可逆又有分析
价值"的数学限制。普通查看 / 同事级别的"看一眼数字猜出公司"已经能防住了。
